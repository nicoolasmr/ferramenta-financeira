# PCI DSS: O Guia para quem não quer ser preso (ou multado)

O **Payment Card Industry Data Security Standard (PCI DSS)** é um conjunto de regras escrito pelas bandeiras (Visa/Mastercard) para garantir que dados de cartão não vazem.

A regra de ouro do PCI para startups é: **Nunca toque no PAN (Primary Account Number).**
Se o número do cartão (16 dígitos) passar pelo seu servidor (mesmo que na memória RAM), você entra no escopo PCI SAQ-D (o mais difícil). Auditoria anual, testes de invasão, firewall físico. É o inferno.

## A Solução: Tokenização de Frontend

Use componentes como **Stripe Elements** ou **RevenueOS Vault**.
O campo de input do cartão não é um `<input>` HTML seu. É um `<iframe>` hospedado diretamente no servidor seguro do gateway.

### O Fluxo Seguro:
1.  Usuário digita o cartão no iframe do RevenueOS.
2.  O dado vai direto do Browser -> RevenueOS.
3.  O RevenueOS devolve um **Token** (`tok_123abc`) para o Browser.
4.  Seu frontend envia o Token para o seu Backend.
5.  Seu Backend usa o Token para cobrar.

Nesse cenário, **seu servidor nunca viu o número do cartão**. Você se qualifica para o questionário PCI SAQ-A (super simples).

## Checklist de Segurança

*   [ ] **HTTPS Forçado:** Nunca trafegue dados em HTTP. Use HSTS.
*   [ ] **Não logue dados sensíveis:** Jamais escreva CVC ou PAN nos logs do servidor.
*   [ ] **Rotação de chaves:** Troque suas chaves de API a cada 90 dias.
