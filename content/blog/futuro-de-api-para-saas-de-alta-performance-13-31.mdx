# Ninguém quer ler sua documentação REST

Desenvolvedores são preguiçosos (e isso é bom).
Eles querem instalar um pacote, digitar `client.` e ver o autocomplete mágicamente funcionar.

Se você oferece apenas uma referência de endpoints cURL, você está perdendo adoção.

## O Poder do OpenAPI (Swagger)

A fonte da verdade da sua API deve ser um arquivo `openapi.yaml`.
A partir dele, você não escreve o SDK na mão. Você gera.

Ferramentas como **Fern**, **Stainless** ou **OpenAPI Generator** criam clientes para Node, Python, Go e Ruby automaticamente.

## A Importância da Tipagem

Em 2026, oferecer um SDK sem tipos TypeScript (`d.ts`) é inaceitável.
Seu cliente quer saber que `amount` é `number` e não `string` antes de rodar o código.

### Exemplo de DX Ruim:
```javascript
api.createCharge({ val: 100 }) // Erro 400 em runtime, parâmetro era 'value'
```

### Exemplo de DX Boa (RevenueOS):
```typescript
await revenue.charges.create({ 
  amount: 1000, 
  currency: 'BRL' // Autocomplete sugere moedas válidas
})
```

## Resiliência Embutida

Seu SDK deve tratar:
*   **Retries:** Se der erro 503, tentar de novo.
*   **Timeouts:** Não travar a thread para sempre.
*   **Paginação:** Iteradores automáticos (`for await const item of list`).
