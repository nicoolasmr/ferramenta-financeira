# O Desafio do Marketplace

O cliente paga R$ 100.
R$ 80 vai para o Prestador de Serviço.
R$ 20 fica com a Plataforma (Take Rate).

Se o dinheiro cair todo na sua conta e você transferir depois, você paga imposto sobre R$ 100 (Bitributação) e assume risco trabalhista.
A solução é o **Split na Origem**.

## O Fluxo de Split

O Gateway recebe os R$ 100 e já divide internamente.
Na sua conta cai R$ 20. Na conta do prestador cai R$ 80.
Fiscalmente, você só emite nota de R$ 20.

## Implementação Node.js

```javascript
const charge = await revenue.charges.create({
  amount: 10000,
  split_rules: [
    { recipient_id: 'seller_123', percentage: 80, liable: true },
    { recipient_id: 'platform_acc', percentage: 20, liable: false }
  ]
});
```

## Chargeback Liability (Responsabilidade)

Quem paga o pato se o cliente contestar?
No `split_rules`, definimos quem é `liable`. Se for o seller, o gateway tira os R$ 100 dele. Se ele não tiver saldo, tira da plataforma.
