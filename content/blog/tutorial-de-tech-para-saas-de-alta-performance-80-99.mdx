# A Curva do Custo Serverless

Lambda é genial: Paga por milissegundo. Se ninguém usar, custo zero.
Perfeito para: Webhooks, Processamento de Imagem, Cron Jobs.

O problema começa quando você tem **Tráfego Constante**.
Se sua API recebe 1.000 requests por segundo 24h por dia, o Lambda vai custar 5x mais que um servidor EC2 ligado o tempo todo.

## O Problema do Cold Start

Em Java ou .NET, o Lambda demora 5 segundos para "acordar". Para uma API de pagamento, isso é inaceitável (Timeout).
Em Node.js/Go/Rust, o cold start é baixo (< 200ms), mas ainda existe.

## Estratégia Híbrida

Use Serverless para tudo que é "Event Driven" (Assíncrono).
Use Containers (Fargate) para sua API Core (Síncrona).
