# "Manda o comprovante no Zap"

Isso não escala.
Se você usa a chave Pix (CNPJ) da empresa e pede para o cliente depositar o valor da fatura, você criou um pesadelo.
Dois clientes depositam R$ 50,00 no mesmo dia. Quem pagou o quê?

## Pix Dinâmico (Cobrança)

A solução é gerar um **QR Code Dinâmico** (ou Pix Copia e Cola) para cada transação.
Esse QR Code carrega um `txid` único.
Quando o cliente paga, o banco notifica via Webhook: "O Pix com txid 123 foi pago".
Você sabe exatamente qual fatura baixar. Irrefutável.

## Pix Saque/Troco e Devolução

Sua API deve suportar devolução automatizada.
Se o cliente cancelar a compra em 7 dias (Lei do Arrependimento), você precisa devolver o Pix instantaneamente via API, não pedindo a chave dele por e-mail.

O RevenueOS trata Pix como cidadão de primeira classe, com conciliação automática via `txid`.
