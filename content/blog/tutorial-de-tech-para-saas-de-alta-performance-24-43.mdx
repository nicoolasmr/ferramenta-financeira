# "Mas na minha máquina funciona"

Em sistemas que lidam com dinheiro, essa frase é inaceitável.
O ambiente de Dev deve ser idêntico ao de Prod.

## Imagens Distroless

Não use `FROM node:18`. Essa imagem tem Debian completo, bash, curl... ferramentas que hackers amam.
Use `FROM gcr.io/distroless/nodejs`.
Ela só tem o binário do Node. Não tem shell. Mesmo se o hacker conseguir RCE (Remote Code Execution), ele não consegue rodar `ls` ou `wget`.

## Multi-stage Builds

Não envie o código fonte TypeScript ou `node_modules` de dev para produção.
1.  **Build Stage:** Instala tudo, compila TS.
2.  **Runtime Stage:** Copia apenas o `dist/` e `package.json` de produção.

Isso deixa sua imagem com 50MB em vez de 1GB e sem segredos vazados.
