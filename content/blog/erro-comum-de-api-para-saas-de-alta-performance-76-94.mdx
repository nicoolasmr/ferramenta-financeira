# GraphQL vs REST: Qual escala melhor para dados financeiros?

Em uma API REST clássica, para pegar as faturas pagas de um usuário, você faz:
1.  `GET /users/1`
2.  `GET /users/1/invoices`
3.  `GET /invoices/X/payments`

Isso se chama **Overfetching** e **Underfetching**.
Em mobile networks, isso significa latência.

Com GraphQL, você faz 1 request:
```graphql
query {
  user(id: 1) {
    invoices {
      amount
      status
      payments {
        method
      }
    }
  }
}
```

## O Problema N+1

O perigo do GraphQL é o desenvolvedor backend esquecer de otimizar.
Se você não usar `DataLoaders`, o servidor vai fazer 1 query no banco para o usuário e N queries para as faturas.

**Regra:** Se sua API for usada por Dashboards pesados (como o RevenueOS Ops), GraphQL é mandatório. Se for usada por webhooks simples, REST é mais robusto.

## Por que o RevenueOS usa Híbrido
