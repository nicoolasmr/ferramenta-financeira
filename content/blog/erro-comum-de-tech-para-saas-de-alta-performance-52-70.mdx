# Juros Compostos no Código

Dívida técnica não é código feio. É código que desacelera o time.
No começo, fazer `if (cliente == 'Nubank')` resolve.
Dois anos depois, tem 500 IFs espalhados. Adicionar uma feature nova leva semanas.

## Quando reescrever?

Nunca faça o "Big Rewrite" (reescrever tudo do zero). Isso mata empresas (Netscape).
Faça o **Strangler Fig Pattern**.
1.  Crie o novo sistema de Billing ao lado do antigo.
2.  Migre 1% dos clientes.
3.  Migre 10%.
4.  Quando todos migrarem, desligue o antigo.

## Dívida Intencional

Às vezes, assumir dívida é estratégico. "Precisamos lançar na Black Friday".
Ok, mas anote a dívida no Backlog: "Refatorar Módulo X (Pós-BF)".
Se você não pagar o principal, os juros (bugs e lentidão) vão comer seu budget de engenharia.
