# O Mistério do Erro 500

O cliente reclama: "Deu erro".
Você olha o log: `Error: Connection lost`.
Onde? No frontend? No backend? No banco? No serviço de e-mail?

Sem **Distributed Tracing**, você está voando às cegas.

## Os 3 Pilares da Observabilidade

1.  **Logs (Eventos):** "Ocorreu um erro X no timestamp Y". Use JSON estruturado, não texto puro.
    *   Ex: `{"level": "error", "user_id": 123, "msg": "Payment failed"}`
2.  **Métricas (Agregados):** "A CPU está em 80%", "Temos 50 falhas por minuto". Use Prometheus/Datadog.
3.  **Tracing (Contexto):** O caminho do request.
    *   O request recebe um `Trace-ID: abc-123` no Nginx.
    *   Esse ID é passado pro Node.js -> Postgres -> Redis -> Stripe.
    *   Você visualiza um gráfico de cascata (Waterfall) mostrando onde demorou.

## OpenTelemetry

Não se prenda a vendor lock-in. Use **OpenTelemetry (OTel)**. É o padrão da indústria para coletar esses dados.
Você instrumenta seu código uma vez e pode enviar os dados para DataDog, New Relic, Grafana ou ElasticSearch.
