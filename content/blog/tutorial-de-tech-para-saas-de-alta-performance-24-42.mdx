# O Framework não é seu amigo

Seu código de negócio (`CalcularJurosService`) não deve importar `express` ou `typeorm`.
Se importar, você casou com a biblioteca.

## A Regra de Dependência

**Entities (Core) < Use Cases < Controllers < Frameworks (External)**
A seta da dependência sempre aponta para dentro. As camadas internas não sabem nada sobre as externas.

## Exemplo: Transferência

```typescript
// Core (Enterprise Logic)
class Transfer {
  constructor(accountFrom, accountTo, amount) { ... }
}

// Application (Use Case)
class ExecuteTransfer {
  constructor(repository) { this.repo = repository; } // Inversão de Dependência
  async execute(input) { ... }
}

// Adapter (Infrastructure)
class PostgresAccountRepo { ... }
```

Isso permite que você troque o banco de dados (Postgres -> Mongo) ou a web (Express -> Fastify) sem tocar em uma linha de lógica financeira.
