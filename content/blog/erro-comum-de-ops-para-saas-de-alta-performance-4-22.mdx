# O Problema da Dupla Verdade

O CRM foi feito para ser flexível (Vendas quer editar tudo).
O Billing foi feito para ser rígido (Contabilidade exige precisão).

Quando você tenta sincronizar os dois manualmente, erros acontecem.
*   Vendedor altera valor da oportunidade no CRM, mas não avisa o financeiro.
*   Cliente faz upgrade no site (Self-service), mas o CRM não sabe.

## Direção da Sincronização

A regra de ouro: **Financeiro é o Mestre (Master). CRM é o Espelho (Slave).**

O contrato financeiro é a verdade legal.
Qualquer alteração no CRM deve ser apenas uma "Solicitação de Mudança" que, se aprovada, altera o Billing.
Qualquer alteração no Billing (ex: Pagamento Falhou) deve ser empurrada para o CRM imediatamente.

## Campos Calculados

Não peça para o vendedor preencher "LTV" ou "Churn Risk" manualmente.
O RevenueOS calcula esses campos baseados no histórico real de pagamentos e injeta no CRM via API.
Seu time de vendas sempre vê dados frescos.
