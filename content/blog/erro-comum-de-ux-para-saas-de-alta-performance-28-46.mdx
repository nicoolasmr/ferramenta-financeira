# Abaixo de 100ms

100ms é o limite onde a ação parece "instantânea" para o cérebro humano.
Se sua API leva 500ms, o app parece "travado".

## Optimistic UI (UI Otimista)

Quando o usuário clica em "Salvar", você muda a UI para "Salvo!" imediatamente, **antes** da resposta do servidor.
Se o servidor der erro 2s depois, você reverte e mostra um toast "Falha ao salvar".
Em 99% das vezes, dá certo, e a experiência é mágica.

## Skeleton Screens

Não mostre um spinner girando numa tela branca. Isso aumenta a ansiedade.
Mostre o esqueleto da página (retângulos cinzas pulsando). Isso dá a sensação de "o conteúdo já está aqui, só falta o texto".
