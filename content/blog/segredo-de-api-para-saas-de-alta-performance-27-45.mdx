# O Modelo "Castelo e Fosso" falhou

Antigamente, você tinha um Firewall na borda. Passou dele, a rede interna era aberta.
Se um hacker comprometer seu servidor Nginx, ele tem acesso livre ao Banco de Dados?

Se a resposta for sim, você está vulnerável.
No modelo **Zero Trust**, nós assumimos que a rede interna já está comprometida.

## Princípios Chave

1.  **Autenticação Mútua (mTLS):** O Serviço A só fala com o Serviço B se ambos apresentarem certificados TLS válidos.
2.  **Least Privilege (Menor Privilégio):** O Worker de E-mail tem credenciais de banco que permitem APENAS ler a tabela `emails`. Ele não consegue ler `credit_cards`.
3.  **Criptografia em Repouso e em Trânsito:** Tudo é criptografado. Mesmo dentro da VPC da AWS.

## Rotação de Credenciais

Suas senhas de banco de dados devem rodar automaticamente a cada X horas.
Nós usamos vaults (como HashiCorp Vault ou AWS Secrets Manager) para injetar credenciais temporárias nos containers.
Nunca commite `.env` no Git.
