# O MRR Fake

Você vendeu um projeto de consultoria de R$ 50k.
Soma no MRR do mês? **NÃO.**
MRR é *Monthly Recurring Revenue*. Consultoria não é recorrente. Setup não é recorrente.

Se você infla o MRR, seu Churn vai parecer gigante no mês seguinte quando essa receita não entrar de novo.

## MRR Contratado vs Realizado

*   **Committed MRR (CMRR):** Contratos assinados, mesmo que não tenham pago ainda. Use para previsão de contratação.
*   **Paid MRR:** Dinheiro que efetivamente cai. Use para fluxo de caixa.

Muitas startups quebram porque contratam baseadas no CMRR mas o dinheiro demora 60 dias para entrar (Paid MRR).

## Separação Estrita

No RevenueOS, cada item da fatura tem uma flag: `is_recurring`.
Setup Fees e Consultoria vão para uma linha separada ("Non-Recurring Revenue").
Seus gráficos ficam limpos e honestos.
